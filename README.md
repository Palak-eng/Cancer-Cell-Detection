# 🎗️ Cancer Cell Detection

An AI-assisted histopathology image classifier that predicts whether a cell
image is **Benign** or **Malignant**, built with a transfer-learning
EfficientNetB0 model and deployed as an interactive Streamlit app.

> ⚠️ **Disclaimer:** This is a learning / portfolio project, **not a medical
> diagnostic tool**. Predictions should never be used for real clinical
> decisions. Always consult a qualified pathologist or physician.

---

## 🔍 Overview

- **Task:** Binary image classification (Benign vs. Malignant)
- **Architecture:** EfficientNetB0 (pretrained on ImageNet) + custom
  classification head, via transfer learning
- **Input:** 224 × 224 RGB histopathology images
- **Interface:** Streamlit web app with upload → predict → confidence score

## 📊 Dataset

This project uses the
[Cancer Detection Dataset](https://www.kaggle.com/datasets/mani11111111111/cancer-detection-dataset/data)
from Kaggle.

To retrain the model yourself:

1. Download the dataset from the link above.
2. Extract it into the project root so the structure looks like:
   ```
   Cancer_Dataset/
   ├── train/
   │   ├── benign/
   │   └── malignant/
   └── test/
       ├── benign/
       └── malignant/
   ```
3. Run `python train.py` (see [Training](#-training) below).

## 🏋️ Training

`train.py` trains an EfficientNetB0-based classifier on the dataset described
above.

```bash
python train.py
```

This will:
- Load and augment images from `Cancer_Dataset/train` (80/20 train/validation split)
- Fine-tune a classification head on top of a frozen EfficientNetB0 backbone
- Save the best checkpoint to `models/best_model.keras`
- Save the final trained model to `models/final_model.keras`

**Key training details:**

| Setting | Value |
|---|---|
| Base model | EfficientNetB0 (frozen, ImageNet weights) |
| Input size | 224 × 224 RGB |
| Loss function | Binary Crossentropy |
| Optimizer | Adam |
| Callbacks | EarlyStopping, ModelCheckpoint, ReduceLROnPlateau |

> **Note:** EfficientNetB0 normalizes pixel values internally. Do **not**
> manually rescale images to 0–1 before feeding them into the model — this
> was a bug in an earlier version of this project that caused the model to
> output ~50% confidence on every prediction.

## 🚀 Running the App Locally

Once you have a trained model at `models/best_model.keras`:

```bash
streamlit run app.py
```

Then open the local URL Streamlit prints (usually `http://localhost:8501`).

### Using a custom logo

Drop a PNG/JPG into `assets/logo.png` and the app will use it automatically
in the sidebar and page headers. If no file is found, it falls back to a 🎗️
emoji.

## ⚠️ Known Limitations

- **No out-of-distribution (OOD) detection.** The model was trained only on
  two classes — benign and malignant histopathology tissue — and has no
  concept of "this isn't a cell image at all." Feeding it unrelated images
  (logos, photos, random graphics, etc.) will still produce a confident-looking
  prediction that is meaningless, since the model has no rejection mechanism
  for inputs outside its training distribution.
- **Low confidence on some real samples.** Predictions in the 50–65% range
  indicate genuine model uncertainty rather than a reliable call, and should
  be treated with caution.
- **Not clinically validated.** This model has not been evaluated against a
  labeled, held-out clinical test set, and its real-world accuracy is
  unknown. See the [Disclaimer](#-cancer-cell-detection) at the top of this
  README.

**Possible future improvements:** add a third "not applicable / non-tissue"
training class, or a separate OOD/confidence-threshold pre-check before
returning a prediction.
